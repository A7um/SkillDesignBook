# Skill 设计之书

**如何写好 SKILL.md** — 数据来自 **ClawHub 下载量前 1,000 的 Skill**（累计约 1,250 万次下载），从中归纳出的设计哲学、写作方法和常见模式。

---

## 这本书讲什么

写给要写 SKILL.md、又要让 Agent 在长任务里跟得住的人：实用、有数据支撑。书里的原则和模式，都来自对真实生产 Skill 的直接阅读，例如：

- **OpenAI** — 14 个 Skill（Agents SDK + 精选目录）
- **Anthropic、Vercel、Stripe、Sentry、Trail of Bits** — 18 个官方 Skill
- **Addy Osmani** — 19 个偏工程生命周期的 Skill
- **ClawHub 社区** — 下载量前 1,000（约 1,250 万次下载；其中 990 个对上了 SKILL.md，另有 889 个脚本被扫过）
- **整库** — 共 58,593 个 Skill 做过程序化统计

## 怎么读

**第一部分 — 设计哲学** 从 Skill 自己的话里抠立场：「核心原则」「核心哲学」「方法论」「为什么存在」，以及「绝不」「始终」「把……当作……而不是……」这类句子。每条原则都挂到具体 Skill 的原文。

**第二部分 — 怎么写** 按 SKILL.md 的结构拆开讲：Description（相当于路由边界）、Body（章节顺序、语气、决策树、输出模板、脚本和约束）、以及从语料里归纳出的 **16 种写法模式（A–P）**、文件怎么摆、怎么测。

**第三部分 — 数据侧写** 把数字摊开：下载前 20、整体统计、前 1,000 与全库的对比、脚本语言占比（Python 约 56%、Bash 约 32%、JS/TS 约 12%），并单独拆 **`self-improving-agent`**（约 39.8 万次下载，ClawHub 第一）这个案例。

**参考资料** — 来源目录，110+ 条可点的链接，方便你回到原始 Skill、博文和论文。

## 几条阅读路线

- **「我要写第一个 Skill」** → 第 1 章 → 第 2 章（Description）→ 第 3 章（Body）→ 第 4 章（挑模式）→ 第 5 章（目录与测试）
- **「我只想看设计原则」** → 读第 1 章即可
- **「我想对照真实 Skill」** → 第 4 章（模式 A–P）+ 第 6 章（统计）
- **「我想查某个出处」** → [来源目录](14-source-catalog.md)

## 一句总纲

> Skill 不是文档。文档在说明「是什么」；Skill 在表态：默认怎么做、哪些路不要走、会踩哪些坑、该怎么想这件事。排在前面的 Skill，往往都写得有棱角 — 排名也在奖励这种写法。
