# Skill 设计之书

**如何编写 SKILL.md** — 从 **ClawHub 下载量前 1,000 名的 Skill**（总计 1250 万次下载）中实证提炼出的设计哲学、方法论与模式。

---

## 这本书是什么

一本实用的、以数据为支撑的 SKILL.md 编写指南，面向长时运行的 AI Agent 场景。本书中每一条原则和模式都源自对生产级 Skill 的直接分析：

- **OpenAI** — 14 个 Skill（Agents SDK + 精选目录）
- **Anthropic、Vercel、Stripe、Sentry、Trail of Bits** — 18 个官方公司 Skill
- **Addy Osmani** — 19 个生命周期工程 Skill
- **ClawHub 社区** — 下载量前 1,000（1250 万次下载，990 个匹配的 SKILL.md 文件，889 个脚本文件被分析）
- **完整语料库** — 58,593 个 Skill 被程序化分析

## 全书结构

**第一部分 — 设计哲学** 从 Skill 自身的"核心原则"/"核心哲学"/"方法论"/"为什么存在"等章节中挖掘出 10 条设计原则，以及它们明确的"绝不……"/"把 X 当作 Y，而不是 Z"/"这不是……"声明。每条原则都直接引用自具体的顶级 Skill。

**第二部分 — 写作方法论** 逐步讲解 SKILL.md 文件各部分的写法：Description（路由决策边界）、Body（章节排序、指令风格、决策树、输出模板、脚本与散文、约束语言）、Skill 模式（从语料分析中发现的 16 种结构性原型 A–P）、文件架构（目录结构、脚本、引用、测试）。

**第三部分 — 实证分析** 呈现数据：下载量前 20 的 Skill、总体统计、模式在前 1,000 与完整语料库中的频率对比、脚本分析（Python 56%、Bash 32%、JS/TS 12%），以及 `self-improving-agent`（39.8 万次下载 — ClawHub 排名第一）的案例研究。

**参考资料** — 来源目录，包含 110+ 超链接引用，指向所有被引用的 Skill、博客文章和论文。

## 阅读路径

- **"我想写我的第一个 Skill"** → 第 1 章（哲学）→ 第 2 章（Description）→ 第 3 章（Body）→ 第 4 章（选择模式）→ 第 5 章（文件架构与测试）
- **"我想理解设计原则"** → 仅第 1 章
- **"我想看真实 Skill 的模式"** → 第 4 章（模式 A–P）+ 第 6 章（数据分析）
- **"我想查找特定的 Skill"** → [来源目录](14-source-catalog.md)

## 元原则

> Skill 不是文档。文档描述事物是什么；Skill 做出主张。它声明默认值、拒绝替代方案、命名失败模式、纠正模型的心智模型。排名靠前的 Skill 都是有态度的 — 排名机制奖励这一点。
